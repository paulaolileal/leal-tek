Clean code não é só para desenvolvedores humanos.

Agentes de IA geram código — e esse código pode ser limpo ou uma bagunça, dependendo das instruções que você deu. Os princípios são os mesmos: nomes significativos, funções com responsabilidade única, sem repetição, boa legibilidade.

A diferença é que, com agentes, você pode codificar essas expectativas explicitamente.

Uma prática que adotei: um arquivo `AGENTS.md` na raiz do projeto com instruções de codificação claras:

```
- Do not create comments not useful
- Name variables with significant names
- Avoid repeating code
- Write code with good spacing and readability
- Functions must have only one responsibility
```

O resultado é consistentemente melhor. O agente segue o padrão do projeto, não o seu padrão genérico de treinamento.

Clean code para agentes é Developer Experience para o seu futuro eu.
