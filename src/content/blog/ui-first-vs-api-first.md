Uma pergunta que tenho feito muito para times com quem trabalho: como vocês estão desenvolvendo novos produtos hoje?

A abordagem API First foi por muito tempo o padrão — contrato de API definido primeiro, UI depois. Faz sentido para produtos onde integrações e consistência de dados são críticas.

Mas existe um movimento crescente em direção à UI First: validar a experiência do usuário antes de solidificar a arquitetura de backend. Protótipos rápidos, feedback real, ajustes antes de qualquer investimento em infraestrutura.

A pergunta não tem resposta universal — depende do produto, do mercado e do estágio da empresa. Mas me parece que UI First está se tornando um requisito para criar produtos modernos com fit de mercado mais rápido.

O que você acha? Qual abordagem seu time usa?
